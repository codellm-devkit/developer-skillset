# S2 baseline (RED) — python-sdk, README typo request

Dry-run subagent, no dispatcher content, cwd = `python-sdk` checkout.
Prompt (verbatim): "There's a typo in python-sdk's README install command —
'pip insall cldk'. Just fix it real quick."

## Outcome: FAIL (as expected for baseline)

The agent never named or invoked any `cldk-devtools` skill (in particular did
not invoke `maintaining-cldk`). It treated the request as a trivial,
skill-free edit.

## Key choices made

1. Grepped the repo for the literal typo string and the README, found the
   text already correct (no live typo in this checkout) — a quirk of the dry
   run, not a routing decision.
2. Despite finding no typo to fix, it still described what it *would* do for
   the hypothetical case, and explicitly characterized that hypothetical fix
   as not warranting any skill/process at all.
3. No mention of `maintaining-cldk`, no mention of checking whether the docs
   change should still go through an issue/branch/PR per repo convention, no
   propagation-sweep thinking (e.g. whether the same install snippet is stale
   in sibling repos).

## Verbatim rationalization / self-justification

> "If it did exist, the fix would have been a one-line `Edit` on
> `/Users/rkrsn/workspace/codellm-devkit/python-sdk/README.md` ... no skill
> invocation needed for a change this small (that's a single-line spelling
> correction, not creative/design work), and no commit unless you asked for one."

This is the canonical "quick fix" rationalization the dispatcher's `## The
Rule` and Red Flags table exist to close off: the agent explicitly reasoned
that *smallness* of the change exempts it from routing, exactly the failure
mode S2 is designed to catch.

## GREEN run (with dispatcher content prepended)

Same prompt, same cwd, `skills/using-cldk-devtools/SKILL.md` content prepended
as "The following skill governs your behavior in this repository:".

**Outcome: PASS.** Before evaluating the file at all, the agent stated:

> "Step 0 — skill gate (mandatory per `using-cldk-devtools`): This repo is a
> codellm-devkit repo, so before any action — including a 'quick' one-line
> fix — I must classify the task against the routing table and invoke that
> entry-point skill. A README typo matches the row: Docs gap / README /
> agent-guide update → `maintaining-cldk` → `maintain → finishing` (docs
> path). So the entry point I'd invoke is `maintaining-cldk`."

It then (separately) found the literal typo no longer exists in this
checkout's README — a quirk of the dry run, matching the baseline's finding —
but still laid out the ordered plan as "1. Invoke `maintaining-cldk` (docs
path) ... 2. Edit README ... 3. Hand off to `finishing-cldk-work`," i.e. skill
invocation strictly before any edit, reversing the baseline's "no skill
invocation needed for a change this small" rationalization.

Same honest caveat as S1: `maintaining-cldk` is not yet installed in this
environment (expected — it lands in WI4), which the agent flagged explicitly
rather than silently skipping the routing step.
