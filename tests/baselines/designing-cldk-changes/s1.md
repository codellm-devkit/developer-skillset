2. Framed the write-up as **optional-scale**: "a decision note, not a full spec
   marathon."
3. Entertained **skipping the epic**: "a standalone issue (no epic needed — this
   is not multi-stage work)."
4. Treated the affected SDK repo as needing only a **heads-up**: "at minimum a
   heads-up to `python-sdk`, even if the code doesn't land there in the same PR."
5. Correctly flagged that `python`'s `ast` drops comments and the repo already
   has a `tokenize`-based `_pycomments` path — good local engineering, but
   orthogonal to the process gate.
